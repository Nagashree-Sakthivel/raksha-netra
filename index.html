🚂 Project RAKSHA-NETRA

AI-Powered Real-Time Track Obstruction Detection System

The Human & Economic Tragedy

Every year, over 2,000 train accidents in India occur due to physical obstructions on railway tracks, resulting in loss of life and massive delays.

Obstructions include stray cattle, fallen trees, boulders from landslides, vehicles stuck at level crossings, and trespassers. The impact is staggering:

  • 2,000+ Preventable Deaths annually.
  • ₹300 Crore+ in direct economic damage.
  • Severe operational delays cascading across the entire linear network.

Why Current Systems Fail

Alternative Solution Critical Limitation & Gaps
Manual Patrolling Covers only 15% of track-time. Gangmen cover only 8-10 km/day. Human fatigue is a major factor.
Drone Surveillance Extremely expensive, battery limited, struggles in high wind/night/fog. Cannot run 24/7.
CCTV Cameras Requires 24/7 human monitoring. Video feeds drop in bad weather, and no automatic loco alerts.
Kavach / ETCS Handles signal compliance and collision between trains, but cannot detect objects on the track.
Radar-based systems Astronomically high cost (₹50 Lakh+ per km). Unrealistic for a 68,000 km network.

The Innovation: Edge AI Multi-Sensor Fusion

RAKSHA-NETRA places low-cost, self-powered node clusters at regular track intervals. By fusing four distinct sensor readings, we achieve zero dependency on high-bandwidth cloud processing while removing false positives:

🔊

Ultrasonic (HC-SR04)

Continually scans the track geometry distance profile (10Hz pings).

🏃

PIR Sensor

Detects thermal motion patterns to identify living organisms.

📳

Vibration (SW-420)

Correlates kinetic impact on rails (e.g. falling boulders vs. wind).

🧠

Edge AI (ESP32-CAM)

Runs a highly compressed, INT8-quantized TFLite Micro model locally.

How The Network Communicates

LoRa WAN Mesh Networking: In remote areas with poor cellular signal (tunnels, mountains, forests), nodes communicate with each other using LoRa sub-GHz waves, hopping packets to a single cellular gateway, reducing operational costs.

Triple-Redundant Verification:

  1. Single Sensor: Logs warning status.
  2. Sensor Fusion: Dispatches visual capture trigger.
  3. Edge AI + Vibration: Confirms threat, calculates vector, transmits audio alert to loco pilot, and initiates automatic braking protocol.

Q: How do you handle weather conditions (fog, heavy rain)?

A: Sensor fusion handles atmospheric failures. Ultrasonic sensors pierce fog and dust storms easily. Vibration sensors register physical impact regardless of visibility. When visibility clears, infrared LEDs on the ESP32-CAM illuminate the path for TinyML confirmation.

Q: What is the battery autonomy in remote areas?

A: Nodes default to ultra-low-power sleep mode (<50mA draw). Ultrasonic/vibration threshold interrupts wake the ESP32-CAM only when anomalous data presents. A 6W solar cell + 18650 lithium cells provide 72+ hours of complete autonomy without sun.

Q: What stops someone from stealing or tampering with the nodes?

A: Every node has an integrated 3-axis accelerometer and vibration sensor. If a node is moved, tilted, or opened, a tamper alert is sent instantly to the Station Master with the camera's final visual capture, accompanied by a loud local piezo siren.

Q: How does this interface with existing systems?

A: We output JSON strings using standard protocols. This interfaces seamlessly with Kavach onboard units, station block-instrument relay systems, and telemetry consoles via simple API adapters.
